Project roadmap, source repositories, packages, and the latest releases for the 5G Media Streaming Reference Tools. Please refer to the [Scope](./scope) page for context on how these components fit together.

## Roadmap

The Kanban board below tracks the planned and in-progress work for the 5G Media Streaming Reference Tools (DASH-based media delivery, Application Server and Application Function components). It is maintained live on GitHub.

[Kanban board: Roadmap](https://github.com/orgs/5G-MAG/projects/48/views/6)

The work is also broken down into individual feature streams, each organised as a Minimum Viable Product (MVP), a self-contained increment that delivers one working capability. For the current status of any item, the linked Kanban board is always authoritative.

| Feature stream | Kanban board |
| --- | --- |
| Basic Media stream handling (MVP#1) | [Board](https://github.com/orgs/5G-MAG/projects/3) |
| Media session handling (MVP#2) | [Board](https://github.com/orgs/5G-MAG/projects/4) |
| M1d Provisioning | [Board](https://github.com/orgs/5G-MAG/projects/8) |
| M3 Link (AF to AS RESTful OpenAPI) | [Board](https://github.com/orgs/5G-MAG/projects/6) |
| Network Assistance and Dynamic Policies | [Board](https://github.com/orgs/5G-MAG/projects/11) |
| QoE metrics collection and reporting | [Board](https://github.com/orgs/5G-MAG/projects/12) |
| Consumption collection and reporting | [Board](https://github.com/orgs/5G-MAG/projects/16) |
| Application Provider User Interface | [Board](https://github.com/orgs/5G-MAG/projects/40) |
| [TS 26.512](https://www.3gpp.org/dynareport/26512.htm) V17.5.0 uplift | [Board](https://github.com/orgs/5G-MAG/projects/18) |
| Cloud infrastructure | [Board](https://github.com/orgs/5G-MAG/projects/21) |
| DVB-I Service using 5G Media Streaming | [Board](https://github.com/orgs/5G-MAG/projects/23) |

## Repositories

The GitHub repositories that make up this project, covering the two server-side roles: the Application Function (AF), which handles provisioning, configuration and reporting, and the Application Server (AS), which caches and delivers the media. Auxiliary repositories are supporting libraries or helper tools rather than primary deliverables.

## Packages

This project also provides the following prebuilt GitHub Container packages: Android library artifacts that client-side applications (such as the 5GMSd-Aware Application) depend on, so you do not have to build the libraries from source.
