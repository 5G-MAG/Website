<h1>5G Core Service Consumers</h1>
</div>
</div>

<div class="topic-lead">
Reference implementation for media applications that consume 5G Core (5GC) network services directly, including policy authorisation, using the interfaces defined for media applications in 3GPP Release 16 and 17.
</div>
<div style={{margin: '8px 0'}}><a class="button button--outline button--primary" href={useBaseUrl('/reference-tools/5g-core/scope')} style={{margin: '2px 4px 2px 0'}}>Scope</a> <a class="button button--outline button--primary" href={useBaseUrl('/reference-tools/5g-core/resources')} style={{margin: '2px 4px 2px 0'}}>Resources</a> <a class="button button--outline button--primary" href={useBaseUrl('/reference-tools/5g-core/tutorials')} style={{margin: '2px 4px 2px 0'}}>Tutorials</a> <a class="button button--outline button--primary" href={useBaseUrl('/reference-tools/5g-core/tutorials#developer-exchange')} style={{margin: '2px 4px 2px 0'}}>Developer Exchange</a></div>

The tools demonstrate how a media application server interacts with the 5G Core (the Policy Control Function (PCF) and Binding Support Function (BSF)) to request network resources dynamically.

This project ships as a set of reusable service consumer libraries, and each library comes with a runnable command-line tool that demonstrates it.

:::tip[Where to start]
- **Scope** for what each service consumer library does and how to choose between them; see [Scope](./scope).
- **Resources** for the source code, repositories and latest releases; see [Resources](./resources).
- **Tutorials** for walkthroughs of testing each service consumer library, starting with libscPCF; see [Tutorials](./tutorials).
:::

**Implemented specifications:** 3GPP [TS 29.214](https://www.3gpp.org/dynareport/29214.htm) (Policy and Charging Control Rx interface), [TS 29.513](https://www.3gpp.org/dynareport/29513.htm) (Policy and Charging function interactions), [TS 26.512](https://www.3gpp.org/dynareport/26512.htm) (5GMS AF interactions with the 5G Core).
