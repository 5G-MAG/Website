New to the Reference Tools? Start with [SDR - HLS Playback over 5G Broadcast](./hls-playback-5gbc), the simplest receiver-only setup. The other tutorials build on the same components. Reference material (sample files, service announcement formats and GPS time sync) is listed at the end of this page.

## [Tutorial: Seamless Switching between Unicast and Broadcast (Android)](./android-mw-seamless-switching)

This tutorial describes the end-to-end setup for seamless switching between 5G Broadcast and unicast delivery on an
Android device. It
utilizes [flute-ffmpeg](https://github.com/5G-MAG/rt-mbms-examples/tree/development/flute-ffmpeg), [rt-libflute](https://github.com/5G-MAG/rt-libflute)
and [rt-mbms-tx-for-qrd-and-crd](https://github.com/5G-MAG/rt-mbms-tx-for-qrd-and-crd) for the transmission of the
media data and the [rt-mbms-mw-android](https://github.com/5G-MAG/rt-mbms-mw-android) for the reception on the client-side.

## [Tutorial: SDR - HLS Playback over 5G Broadcast](./hls-playback-5gbc)

This tutorial provides guidelines how to setup the
Reference Tools to use an HLS stream which was received via 5G Broadcast, processed by
the [MBMS Modem](https://github.com/5G-MAG/rt-mbms-modem) and played via dash.js or hls.js in
the [Web User Interface](https://github.com/5G-MAG/rt-wui).

## [Tutorial: SDR - RTP Playback over 5G Broadcast](./rtp-playback-5gbc)

This tutorial provides guidelines how to setup the
Reference Tools to use an RTP stream which was received via 5G Broadcast, processed by
the [MBMS Modem](https://github.com/5G-MAG/rt-mbms-modem) and played in a third-party media player such as ffplay or
VLC.

## [Tutorial: MBMS Modem with Sample Files](./modem-samplefiles)

This tutorial provides information on the use of the rt-mbms-modem with sample files as input.

## [Tutorial: Docker support for rt-mbms client components (Linux)](./docker-implementation)

The implementation of the rt-mbms processes (5G broadcast receiver) is provided as Docker containers. Details on how to
use the Docker setup can be found in this tutorial.

## [Tutorial: MBMS Modem - Measurements & GPS](./modem-measurements)

This tutorial provides information on the use of the rt-mbms-modem to record data.

## [Tutorial: Configuration of Service Announcement](./configuration-guide)

This tutorial provides an overview of the different configuration options when running the Linux based 5G broadcast
stack. It highlights the required configuration options for the 5G Broadcast Service Announcements files and how to
configure the FLUTE ffmpeg watchfolder implementation for DASH, HLS and seamless switching.

## Reference material

Supporting pages referenced by the tutorials above:

- [Sample Files](../additional/sample-files): recorded I/Q signals for testing without an SDR.
- [Service Announcement Formats](../additional/rt-common-shared/MBMS-service-announcement-files): the three SA file formats and when to use each.
- [GPS Time Synchronization](../additional/rt-mbms-modem/GPS-time-synchronization): disciplining the receiver clock from GPS for measurements.
- [Requirements](./requirements): hardware, SDR and OS requirements, plus validated test configurations.
