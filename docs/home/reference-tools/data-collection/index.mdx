<div class="topic-lead">
Reference implementation of the User Equipment (UE) side data collection, reporting, and event exposure framework defined in 3GPP TS 26.531 and TS 26.532.
</div>
<div style={{margin: '8px 0'}}><a class="button button--outline button--primary" href={useBaseUrl('/reference-tools/data-collection/scope')} style={{margin: '2px 4px 2px 0'}}>Scope</a> <a class="button button--outline button--primary" href={useBaseUrl('/reference-tools/data-collection/resources')} style={{margin: '2px 4px 2px 0'}}>Resources</a> <a class="button button--outline button--primary" href={useBaseUrl('/reference-tools/data-collection/tutorials')} style={{margin: '2px 4px 2px 0'}}>Tutorials</a> <a class="button button--outline button--primary" href={useBaseUrl('/reference-tools/data-collection/tutorials#developer-exchange')} style={{margin: '2px 4px 2px 0'}}>Developer Exchange</a></div>

The tools enable 5G devices to collect, aggregate, and report media performance metrics (playback quality, buffer events, network conditions, and consumption data) to the 5G Media Streaming Application Function (AF). This data feeds network-assisted Quality of Experience (QoE) optimisation, analytics dashboards, and adaptive policy decisions.

**Implemented specifications:** 3GPP TS 26.531 (data collection and reporting framework), [TS 26.532](https://www.3gpp.org/dynareport/26532.htm) (data collection client procedures), [TS 26.512](https://www.3gpp.org/dynareport/26512.htm) (5GMS AF interfaces for data reporting submission). The interfaces these specifications define are referred to on the [Scope](./scope) page by their reference-point names R1 to R6.

:::tip[Where to start]
- **Scope** for the DCAF architecture and the R1 (provisioning), R6 (event exposure) and R2 (direct data reporting) interfaces it implements — see [Scope](./scope).
- **Resources** for the source code, repositories and latest releases — see [Resources](./resources).
- **Tutorials** for the Docker and Insomnia walkthrough that sets up the DCAF and exercises its R1, R6 and R2 REST APIs — see [Tutorials](./tutorials).
:::
