<div class="topic-lead">
Reference implementation for delivering public warning and emergency alert messages over 5G Broadcast infrastructure.
</div>
<div style={{margin: '8px 0'}}><a class="button button--outline button--primary" href="/reference-tools/emergency-alerts/scope" style={{margin: '2px 4px 2px 0'}}>Scope</a> <a class="button button--outline button--primary" href="/reference-tools/emergency-alerts/resources" style={{margin: '2px 4px 2px 0'}}>Resources</a> <a class="button button--outline button--primary" href="/reference-tools/emergency-alerts/tutorials" style={{margin: '2px 4px 2px 0'}}>Tutorials</a> <a class="button button--outline button--primary" href="/reference-tools/emergency-alerts/tutorials#video-library" style={{margin: '2px 4px 2px 0'}}>Video Library</a></div>

The tools enable authorities and broadcasters to distribute alerts to all receivers in a coverage area simultaneously (without requiring a data connection or active SIM) using the Cell Broadcast Service (CBS) transported over LTE-based 5G Broadcast, also known as Further evolved Multimedia Broadcast Multicast Service (FeMBMS). The work builds on top of the 5G Broadcast Hybrid TV/Radio platform and has been demonstrated in broadcast network setups; for the current implementation status, see the caveat on the [Scope](./scope) page.

:::tip[Where to start]
This project extends the base 5G Broadcast transmitter and receiver tools, so it is best read alongside [5G Broadcast: Hybrid TV/Radio](../5g-broadcast). To set it up, follow the [Emergency Alerts end-to-end tutorial](./tutorials/end-to-end).
:::

**Implemented specifications:** ETSI TS 103 720 (5G Broadcast System, including CBS transport), 3GPP [TS 23.041](https://www.3gpp.org/dynareport/23041.htm) (Cell Broadcast Service architecture), [TR 36.976](https://www.3gpp.org/dynareport/36976.htm) (LTE-based 5G Broadcast overview).
