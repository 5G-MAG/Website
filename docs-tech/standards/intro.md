This is the entry point to **5G-MAG's work on Standards and Specifications**. Drawing on its software development, implementation and deployment experience, 5G-MAG provides feedback, requirements and liaison statements to Standards Developing Organisations (SDOs) such as 3GPP and ETSI.

5G-MAG documentation spans three areas: this Standards section (5G-MAG's feedback, requirements and liaison statements to standards bodies), the [Technical Documentation](/tech/intro) section (specification analysis, explainers and videos), and the developer portal at https://developer.5g-mag.com (the open-source Reference Tools). This page covers the Standards section. If you are implementing rather than tracking standards, start with the developer portal.

## Standards Activities

| Activity | Description |
|----------|-------------|
| [Feedback to Specifications](/tech/standards/feedback) | Structured feedback submitted to 3GPP, ETSI and other SDOs based on implementation experience |
| [Requirements towards Standardisation](/tech/standards/requirements) | Members' input consolidated into formal requirements submitted to standards bodies |
| [Liaison Statements & Inputs](/tech/standards/ls) | Formal liaison statements and informative documents provided to SDOs |

For 3GPP work item tracking see [3GPP Issue Tracking](/tech/standards/3gpp-issue-tracking) and [3GPP Work Items per Release](/tech/standards/3gpp-work-items). For acronyms and recurring terms used across this portal, see the [Glossary](/tech/standards/glossary).

## Quick Reference A–Z

The activities above describe how 5G-MAG feeds back to standards bodies; the index below lists the specific work areas that this feedback covers. If you are not sure where to start, 5G Media Streaming, 5G Broadcast and 5G Multicast-Broadcast Services are the most active areas.

| Topic | What it covers |
|-------|----------------|
| [5G Broadcast: Hybrid TV/Radio](/tech/standards/5g-broadcast) | LTE-based, free-to-air linear TV and radio delivery (ETSI TS 103 720) |
| [5G Media Streaming](/tech/standards/5gms) | 3GPP framework for adaptive streaming media over 5G (5GMS) |
| [5G Multicast & Broadcast Services](/tech/standards/5g-mbs) | Native 5G core and NR multicast-broadcast delivery (MBS) |
| [6G](/tech/standards/6g) | Early study of media aspects for 6G systems |
| [AI/ML in 5G Media](/tech/standards/ai-ml) | AI and machine learning applied to 5G media traffic and analytics |
| [Avatar Communications](/tech/standards/avatar) | Avatar representation and immersive real-time communication |
| [Beyond 2D Video](/tech/standards/beyond-2d) | Formats beyond a flat single-camera picture (stereoscopic, multiview, volumetric) |
| [DVB-I over 5G](/tech/standards/dvb-i) | DVB-I service discovery and delivery over 5G bearers |
| [Multimedia Content Delivery](/tech/standards/multimedia) | One-way transport protocols for broadcast and multicast delivery |
| [Network APIs](/tech/standards/network-apis) | Exposing network capabilities to applications (CAMARA and 3GPP) |
| [Non-Public Networks](/tech/standards/npn) | Private and non-public 5G network deployment models |
| [Non-Terrestrial Networks](/tech/standards/ntn) | Satellite and other non-terrestrial 5G delivery |
| [Real-Time Media Communication (RTC)](/tech/standards/rtc) | Low-latency, interactive real-time media over 5G |
| [Time-Sensitive Communications](/tech/standards/tsc) | Deterministic, bounded-latency delivery over 5G |
| [UE Data Collection & Exposure](/tech/standards/data-collection) | Device data reporting and network event exposure |
| [Volumetric Video](/tech/standards/v3c) | Coding and carriage of volumetric and immersive video (V3C) |
| [XR Media Integration](/tech/standards/xr) | Extended reality media delivery over 5G |
